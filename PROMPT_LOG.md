# Prompt Log – Campus Companion

## Prompt 1 – Planning

ROLE: You are a senior web developer and accessibility reviewer.

CONTEXT: I am building a Campus Companion web app for first-year students.

CONSTRAINTS:
- HTML, CSS and JavaScript
- Fictional data only
- Must include at least 4 features
- Accessibility should follow WCAG AA principles
- App should be simple to run locally

TASK:
Create a plan for a student web app with timetable, campus directory, society events, canteen menu, lost and found, helpdesk tickets, reminders, and accessibility settings.

OUTPUT FORMAT:
Provide a feature list, page structure, and improvement suggestions.

SELF-CHECK:
Check if the app meets the assignment requirements.

## Prompt 2 – Generate App

ROLE: You are a senior frontend developer.

CONTEXT: We are building a Campus Companion web app for students.

CONSTRAINTS:
- Use one HTML file
- Include CSS and JavaScript inside the file
- Use fictional data only
- Must include at least 4 required features
- Must be accessible and keyboard friendly

TASK:
Generate the full Campus Companion web app.

OUTPUT FORMAT:
Provide the complete index.html code.

SELF-CHECK:
After the code, check whether the app includes the minimum required features.

## Prompt 3 – Review

ROLE: You are an accessibility reviewer.

CONTEXT: I have created a Campus Companion web app using HTML, CSS and JavaScript.

CONSTRAINTS:
- WCAG AA principles
- Clear navigation
- High contrast option
- Forms must have labels
- Fictional data only

TASK:
Review the app for accessibility, usability, and assignment suitability.

OUTPUT FORMAT:
List issues found and suggested improvements.

SELF-CHECK:
Confirm whether the app is suitable for first-year students.

## Prompt 4 – Improve

ROLE: You are a frontend developer improving a student web app.

CONTEXT: The Campus Companion app already includes timetable, locations, societies, canteen, lost and found, helpdesk, reminders and accessibility settings.

TASK:
Improve the app by adding better visual styling, cards, form feedback, high contrast mode and large text mode.

OUTPUT FORMAT:
Provide the updated code.

SELF-CHECK:
Check that the app still uses fictional data only.
